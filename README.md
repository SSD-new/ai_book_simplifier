📚 War-and-Peace-10GB: Как я заставил нейросеть читать Толстого (и выжить)
![alt text](https://img.shields.io/badge/ML-Llama3--Saiga-blueviolet)

![alt text](https://img.shields.io/badge/GPU-RTX%203080%2010GB-green)
Этот репозиторий — результат борьбы обычного школьника с четырехтомником «Войны и мира» и лимитами видеопамяти моей RTX 3080.
Проблема: Читать 2000+ страниц — долго, а найти внятное краткое содержание по главам (без воды) — на удивление сложно.
Решение: Скормить текст Льва Николаевича нейросети.
Нюанс: 10 ГБ VRAM превращают обычную задачу в увлекательный квест по квантованию и промпт-инжинирингу.
🛠 Техстек
Модель: IlyaGusev/saiga_llama3_8b (самая стабильная для русского языка).
Оптимизация: 4-битное квантование через bitsandbytes (NF4).
Логика: Обработка по главам (скользящее окно не зашло — получился «сломанный телефон»).
Оценка: Gemini 3.1 Flash (в роли строгого учителя литературы).
🧠 Главные фичи и "костыли"
1. Борьба с детоубийством (Character Lore)
Нейросеть упорно пыталась «убить» Николая Ростова при Шенграбене, потому что Толстой слишком красиво описывал его ранение. Чтобы спасти героев, в системный промпт была вшита База Данных Персонажей.
Интересный факт: Плотный промпт создал ситуацию «максимальной энтропии», когда вероятность того, что герой выжил или умер, распределилась ровно 50/50.
2. Рождение Графа Бездаровского
В процессе работы был обнаружен новый герой русской классики — Пьер Бездаровский. Это результат галлюцинации, когда модель смешала фамилию «Безухов» с внутренним монологом Пьера о своей «бездарности». Исправляется только жестким лором и постобработкой.
3. "Впихнуть невпихуемое"
Текст режется не просто по символам, а по словам, чтобы не ломать токенизацию на полуслове. Оптимальный размер куска — 7500 символов.
🚀 Как запустить
Железо: Минимум 10 ГБ VRAM (если у вас меньше — молитесь на 2-битное квантование).
Установка:
code
Bash
pip install torch transformers bitsandbytes accelerate
Подготовка: Положите файл tolstoy_voyna-i-mir.txt в корень.
Запуск:
code
Bash
python summarize.py
Скрипт умеет дозаписывать файл. Если свет выключат или видеокарта решит уйти в отпуск — обработка начнется с последней сохраненной главы.
📊 Результаты
Исходник: ~230 000 слов.
Результат: ~18 000 слов (плотная выжимка фактов).
Точность: По оценке Gemini — около 95% (за вычетом инцидента с Бездаровским и парой перепутанных князей).
⚠️ Дисклеймер
Если ваша учительница литературы спросит, откуда у вас в сочинении взялся «Виконт Болконский» или почему Пьер стал Бездаровским — автор ответственности не несет. ИИ — это инструмент, а Толстой — это вечно.
